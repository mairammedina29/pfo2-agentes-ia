PROMPT BASE UTILIZADO

Actuá como un desarrollador Front-End experto en interfaces UI de videojuegos y Diseñador UX/UI Senior de estéticas Cyberpunk y Arcade de los 80/90. Tu objetivo es generar el código completo, limpio y funcional (HTML con CSS unificado en un solo bloque dentro de la etiqueta <style>) para una Landing Page hiper-animada de una tienda online de videojuegos arcade, consolas retro y coleccionables.

Esta interfaz debe ser viva y dinámica, por lo que debés aplicar de forma estricta las siguientes animaciones modernas, guía de estilos, paleta de colores y restricciones de maquetación.

1. GUÍA DE ESTILOS (CSS)
  Configurá un sistema dinámico para modo noche (por defecto en :root) y modo día mediante la clase `.light-mode` aplicada al body. Implementá exactamente estas variables y tipografías de Google Fonts:

  Tipografías:
   - Títulos, botones, etiquetas y UI: 'Orbitron', sans-serif; (Estilo geométrico, usar siempre en mayúsculas).
   - Bloques de texto, párrafos y datos: 'Share Tech Mono', monospace; (Estilo terminal de comandos).

  CSS - Modo Noche (Cyberpunk/Arcade por defecto):
   - bg-gradient: linear-gradient(135deg, #03001e 0%, #7303c0 50%, #ec38bc 100%);
   - text-title-color: #00F2FE; (Cyan Neón Eléctrico)
   - text-subtitle-color: #FFF200; (Amarillo Arcade)
   - text-desc-color: #E2E8F0; (Blanco grisáceo suave)
   - grid-color: rgba(0, 242, 254, 0.15);
   - grid-glow: 0 0 12px rgba(0, 242, 254, 0.75);
   - color-tomato: #FF0055; (Rosa Láser para efectos hover de acción)

  CSS - Modo Día (High-Tech Sci-Fi - Activo con body.light-mode):
   - bg-gradient: linear-gradient(135deg, #eef2f7 0%, #d9e2ec 100%);
   - text-title-color: #0F172A; (Azul Pizarra Oscuro)
   - text-subtitle-color: #FF5500; (Naranja E-Sports)
   - text-desc-color: #334155; (Gris oscuro)
   - grid-color: rgba(255, 85, 0, 0.2);
   - grid-glow: 0 0 10px rgba(255, 85, 0, 0.6);
   - color-tomato: #FF5500;

2. CORE DE ANIMACIONES Y EFECTOS RETRO  (CSS)
 Para erradicar la apariencia estática, programá los siguientes micro-efectos usando transiciones suaves y keyframes optimizados para GPU (transform y opacity):
    - Efecto Pantalla CRT / Scanlines: Generá una capa pseudo-elemento fija sobre el fondo de la pantalla que use un gradiente lineal repetitivo muy sutil simulando líneas de televisión antigua. Añadí una animación de barrido vertical lento que simule el refresco de pantalla analógico.
    - Texto con Efecto Glitch Neón: El título principal de la Hero Section debe tener un effecto de distorsión de color neón sutil usando sombras de texto cruzadas (text-shadow) que vibren de manera asíncrona mediante keyframes.
    - Efecto "Insert Coin" Parpadeante: Los botones secundarios y las llamadas de atención decorativas deben tener un parpadeo de texto intermitente e irregular (flicker) imitando las bombillas físicas de los gabinetes arcade.
    - Efecto de Levitación y Perspectiva 3D en Tarjetas (Hover): Al pasar el cursor sobre las 3 tarjetas de la Hero Section, estas deben reaccionar elevándose, inclinándose ligeramente en perspectiva 3D (transform: perspective(1000px) rotateX() rotateY()), y aumentando de forma fluida el radio de su resplandor perimetral (--grid-glow).
    - Marquesina de Texto Infinita (Ticker de 8-Bits): Creá una barra horizontal deslizante continua en CSS puro para mostrar noticias de "Nuevos Ingresos Arcade" (ej. "¡NUEVO GABINETE PAC-MAN DISPONIBLE! - INSERT COIN"). Debe deslizarse de derecha a izquierda infinitamente de forma fluida y pausarse al pasar el cursor (hover).

3. MAQUETACIÓN VISUAL Y ESTRUCTURA DE LA LANDING PAGE
 Diseñá una maquetación fluida y totalmente responsiva con CSS Grid/Flexbox (aplicando 'min-width: 0;' en columnas para evitar roturas de texto), que contenga los siguientes bloques semánticos:
    1. Cabecera (Header)
    2. Hero Section (Estructura Dividida en 2 Columnas)
    3. Descripción / Sobre Nosotros
    4. Sección de Servicios o Características
    5. Testimonios o Reseñas
    6. Formulario de Contacto
    7. Pie de Página (Footer)

4. RESTRICCIONES DE DESARROLLO 
    - Entregá TODO el código integrado en un único bloque de código legible (HTML con CSS embebido en <style> en el head).
    - No utilices frameworks externos. Todo debe ser CSS nativo puro.
    - Generá el código 100% finalizado y listo para visualizar.
